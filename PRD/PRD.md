## 1. 项目背景

### 1.1 问题陈述

传统帽子电商内容生产面临三大核心痛点：

1. **制作周期长**：预约模特 → 场地拍摄 → 后期修图，全程耗时 2-3 天
2. **成本高昂**：单次拍摄成本数千元，多款式试错成本高
3. **反馈迭代慢**：效果不满意需重新拍摄，时间资金双浪费

### 1.2 项目目标

利用 AI 多模态生图能力，构建**端到端分钟级**的内容生产闭环：

| 目标 | 指标 |
|------|------|
| 效率提升 | 从 2-3 天压缩至 < 10 分钟（4000x+） |
| 成本降低 | 反馈迭代成本降低 ~80% |
| 质量可控 | 风格一致性高，满足小红书头部博主审美标准 |

### 1.3 目标用户

- **主要用户**：帽子电商商家/品牌方
- **使用场景**：新品上架、季末促销、多平台内容分发
- **内容渠道**：小红书、抖音、淘宝详情页

## 2. 功能需求

### 2.1 核心功能列表

| ID | 功能名称 | 优先级 | 描述 |
|----|----------|--------|------|
| F1 | 商品图上传 | P0 | 支持上传帽子正面、侧面参考图 |
| F2 | AI 生图 | P0 | 自动生成 8 张同模特、同场景、多角度模特图 |
| F3 | AI 生视频 | P0 | 自动生成 5 个 3s 短视频 |
| F4 | 风格配置 | P1 | 支持自然语言配置服装风格、场景、拍摄角度 |
| F5 | 飞书同步 | P0 | 一键批量上传至飞书多维表格 |
| F6 | 反馈迭代 | P1 | 支持基于用户反馈优化 Prompt |

### 2.2 详细功能规格

#### F1: 商品图上传

**输入要求**：
- 必填：帽子正面图（{{front_image}}）
- 必填：帽子侧面图（{{lateral_image}}）
- 图片格式：JPG/PNG，建议分辨率 > 1024px

**处理逻辑**：
- 系统自动提取帽子特征（版型、形状、颜色）
- 特征向量传递给生图模型，确保 100% 还原

#### F2: AI 生图

**输出规格**：
- 数量：8 张/组
- 尺寸：3:4 竖版
- 风格：小红书头部博主审美
- 模特特征：中国面孔，28-32 岁，参考卢昱晓长相、高圆圆气质

**场景默认配置**：

| 场景类型 | 场景描述 | 服装要求 |
|----------|----------|----------|
| 居家 | 法式奶油风室内（米白墙面 + 木质地板） | 舒适家居服 |
| 户外 | 高级咖啡馆 / 美术馆 | 经典款大衣/西装 |
| 滑雪场 | 雪道场景 | 专业滑雪服 |

**动作要求**：
- 至少 4 张对镜自拍（手持复古 CCD / 手机）
- 支持坐姿托腮、蹲姿、站姿叉腰等多角度展示
- 同一组图只允许使用一种道具

#### F3: AI 生视频

**输出规格**：
- 数量：5 个/组
- 时长：3s/个
- 格式：MP4
- 内容：基于生图结果生成的动态短视频

#### F4: 风格配置

**支持的自然语言配置项**：

| 配置项 | 示例输入 |
|--------|----------|
| 场景 ({{scene}}) | "居家喝咖啡" / "美术馆看展" / "滑雪场" |
| 服装 ({{clothing}}) | "驼色羊绒大衣" / "白色高领毛衣 + 格纹裙" |

#### F5: 飞书同步

**数据结构**：

| 字段 | 说明 |
|------|------|
| 商品名称 | 用户输入 |
| 场景类型 | 居家 / 户外 / 滑雪场等 |
| 生成时间 | 自动记录 |
| 图片链接 | 8 张图 URL |
| 视频链接 | 5 个视频 URL |

#### F6: 反馈迭代

**迭代机制**：
```
用户反馈 → Prompt 优化 → 功能升级
```

**历史优化记录**：

| 版本 | 优化内容 | 效果 |
|------|----------|------|
| v1.0 | 初始版本 | - |
| v1.1 | 增加场景自适应扩写 | 场景真实度提升 |
| v1.2 | 细化动作指导（至少 4 张对镜自拍） | 帽子展示更完整 |

## 4. 技术方案

### 4.1 系统架构

```
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│  用户前端   │ ───> │  Coze 工作流│ ───> │  飞书表格   │
│ (商品图上传)│      │  (生图引擎) │      │  (结果存储) │
└─────────────┘      └─────────────┘      └─────────────┘
                           │
                           ▼
                    ┌─────────────┐
                    │ Seedream 4.5│
                    │ (生图模型)  │
                    └─────────────┘
```

### 4.2 模型选型

| 模型 | 技术底座 | 测试结果 | 决策 |
|------|----------|----------|------|
| Nano Banana | Gemini 2.5 Flash Image | 画风像是电影 | ❌ |
| 豆包 Seedream 4.5 | 字节自研 | **更符合国人审美** | ✅ |

**A/B 测试结论**（客户反馈）：
> "还是豆包 Seedream 4.5 生成的效果更好。"

### 4.3 Prompt 工程

详见 `/prompts/` 目录下的完整 Prompt 模板。

**核心设计原则**：
1. **结构化约束**：按优先级分层（最优先 → 要求 → 禁止）
2. **风格锚定**：用明星参考确保输出稳定
3. **场景自适应**：支持多场景扩写逻辑
